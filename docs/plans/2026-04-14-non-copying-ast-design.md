# Non-Copying AST Design

**Date:** 2026-04-14
**Status:** Design approved, ready for implementation

## Goal

Make Recursa ASTs borrow from the input string instead of owning copies.
Parsing produces `Cow::Borrowed`; the `Owned` variant exists as an escape
hatch for future AST mutation but is not used by the parser itself.

## Motivation

Every captured token type generated by the `literals!` macro currently
holds a `String`, forcing an allocation per token during parsing. Replacing
`String` with `Cow<'input, str>` and emitting `Cow::Borrowed(matched)`
eliminates those allocations for the common parse-and-read path, while
keeping the option to own the data later.

## Alternatives Considered

**Attribute macro that rewrites `String` → `Cow<'input, str>`.** Rejected.
Derive macros in Rust cannot modify the annotated item — only attribute
macros can — so this would require a new `#[recursa::ast]` wrapper. More
importantly, only ~5 hand-written AST sites contain a `String`; the real
labor is propagating `'input` through every type that transitively holds
an identifier, and no macro can automate that.

**Introduce a `Str<'i>` type alias.** Rejected (YAGNI). `Cow<'input, str>`
appears in maybe 5 hand-written sites; the rest is inside macro output.
The alias adds indirection without meaningful savings.

**Borrow at a higher level and keep leaves as `String`.** Rejected. Does
not avoid the allocations, which are the whole point.

## Framework Changes (recursa-core, recursa-derive)

### `literals!` macro
Change generated leaf from:
```rust
pub struct $name(pub String);
```
to:
```rust
pub struct $name<'input>(pub std::borrow::Cow<'input, str>);
```
Update its `FormatTokens` impl to clone the `Cow` contents.

`keywords!` and `punctuation!` generate unit structs with no captured
text and need no change.

### `recursa-derive/src/parse_derive.rs:220`
Change `#name(matched.to_string())` to
`#name(::std::borrow::Cow::Borrowed(matched))`. Verify `matched` is
already `&'input str` in the surrounding generated code; if not, thread
the lifetime through.

The derive already handles struct generics correctly: it reads
`input.generics`, calls `split_for_impl()`, and picks up a declared
lifetime via `generics.lifetimes().next()` (parse_derive.rs:199). Structs
declared as `Foo<'input> { ... }` will derive cleanly.

### `Visit for String`
`recursa-core/src/visitor.rs:158` has `impl AsNodeKey for String` and
`impl Visit for String`. Replace with impls for `Cow<'_, str>`. Leaf
visit behavior (terminal, no recursion) is identical. `AsNodeKey` keys
on the string contents via `Cow`'s `Deref<Target = str>`.

## pg-sql AST Changes

### Lifetime propagation rule
If a type's fields transitively reference a `'input`-carrying type, it
gains `<'input>`. Types built purely from keywords and punctuation
(e.g. `IsTrue`, `NotNull`) stay lifetime-free. In practice, nearly every
AST type in `pg-sql/src/ast/` will carry `'input` because identifiers
appear almost everywhere.

Approach: file-by-file, leaves first, outward. Compiler errors drive
propagation — each missing `'input` surfaces as a concrete next target.

### Hand-written `String` sites
1. **`pg-sql/src/tokens.rs`** — `UnquotedIdent(String)`,
   `BareAliasName(String)`, `RestOfLine(String)`. Change to
   `Cow<'input, str>`; update `FormatTokens` impls.
2. **`pg-sql/src/ast/mod.rs:219`** — `DollarStringLit { text: String }`.
   Has a manual `Parse` impl that scans for `$tag$ ... $tag$`. Change
   to `text: Cow<'input, str>` and borrow `&input[start..end]`.
3. **`pg-sql/src/ast/mod.rs:352`** — `RawLines(String)`. Change to
   `RawLines(Cow<'input, str>)`; borrow from input.
4. **`pg-sql/src/ast/mod.rs:410`** — `raw_buf = String::new()` accumulator.
   Inspect during implementation: if concatenated lines are contiguous
   in the source, replace with a single `Cow::Borrowed` slice;
   otherwise keep `Cow::Owned(String)`. Field type becomes
   `Cow<'input, str>` either way.

### Out of scope
`main.rs`, `harness.rs`, `formatter.rs`, `bin/gen_stress.rs` — these are
tools/drivers, not AST. Their `String` usage stays.

## Execution Plan

### Worktree setup
1. `git fetch origin && git checkout main && git pull --ff-only`
2. Create worktree via `cipherpowers:using-git-worktrees` on branch
   `feature/non-copying-ast` from updated `main`.
3. Verify `cargo build` and `cargo test` are green before edits.

### Implementation order (bottom-up, compile at each step)
1. `literals!` macro → `Cow<'input, str>`. `cargo check -p recursa-core`.
2. `Visit for String` → `Visit for Cow<'_, str>`. `cargo check -p recursa-core`.
3. `parse_derive.rs:220` → `Cow::Borrowed(matched)`. `cargo check -p recursa-derive`.
4. `pg-sql/src/tokens.rs` leaves → `<'input>`. Expect cascade.
5. `pg-sql/src/ast/*` file-by-file, driven by compiler errors.
6. `DollarStringLit` manual `Parse` impl.
7. `RawLines` + `raw_buf` contiguity decision.
8. Update tests that literal-construct string fields to use
   `Cow::Borrowed("...")` or `.into()`.

### Verification gates
- `cargo build --workspace` after each step.
- `cargo test --workspace` after step 8.
- `cargo clippy --workspace` before commit.
- Optional sanity check: parse-throughput benchmark on one stress file
  before/after to confirm allocations were removed.

### Commit shape
- Steps 1–3: single framework-change commit.
- Steps 4–7: pg-sql commits, ideally one per file or logical group.
- Step 8: tests commit.

### Rollback
If step 5 cascades unmanageably, abandon the worktree; `main` is
untouched.

## Risks
- **Choice-enum codegen** (`derive_choice_enum`, parse_derive.rs:430)
  must handle `ty_generics` consistently during variant construction.
  Verify early during step 1–3.
- **`raw_buf` contiguity** is unknown until inspected; may force
  `Cow::Owned` in that one spot, which is fine.
- **Test construction sites** may be numerous; plan to grep-and-replace
  `String(...)` / `.to_string()` constructions in tests.
